# Task 14 - Linux Server Hardening & Secure Configuration

## Objective
To secure a Linux server by applying hardening techniques and reducing attack surface.

## Steps Performed

- Reviewed system users and services
- Removed unused accounts
- Restricted sudo access
- Disabled root SSH login
- Configured SSH key-based authentication
- Updated system packages
- Enabled automatic security updates
- Configured UFW firewall
- Disabled unnecessary services
- Secured file permissions
- Reviewed authentication logs

## Tools Used
- Ubuntu Linux
- UFW Firewall
- SSH
- Systemctl

## Outcome
Successfully hardened Linux server by implementing secure configurations and best practices.
